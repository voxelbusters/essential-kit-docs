# Documentation Style and Conventions

## Writing Style
- **Tone**: Professional, developer-focused, instructional
- **Audience**: Unity mobile game developers
- **Approach**: Tutorial-based with progressive complexity
- **Voice**: Direct, practical, solution-oriented

## Content Structure Conventions
- **Headers**: Use descriptive, action-oriented titles
- **Code Examples**: Always include practical, runnable code snippets
- **Platform Notes**: Clearly distinguish iOS vs Android specific information
- **Prerequisites**: Listed at beginning of tutorials
- **Learning Path**: Progressive structure from basic to advanced

## Markdown Conventions
- **GitBook Extensions**: Uses `{% embed %}`, `{% hint %}`, `{% content-ref %}` syntax
- **Code Blocks**: Language-specific syntax highlighting (csharp, javascript, etc.)
- **Emphasis**: Uses `<mark>` tags for highlighting key concepts
- **Links**: Internal linking using relative paths
- **Tables**: Used for structured information like properties and settings

## Documentation Patterns
- **Feature Overview**: What it does, why use it, common use cases
- **Setup Instructions**: Platform-specific configuration steps
- **Usage Examples**: Practical code implementation patterns  
- **Testing Guidance**: How to validate functionality during development
- **FAQ Format**: Q&A style for common issues
- **Best Practices**: Bulleted recommendations and guidelines

## Technical Writing Standards
- **API References**: Consistent method/property documentation
- **Error Handling**: Always include error case examples
- **Platform Differences**: Explicit callouts for iOS/Android variations
- **Code Comments**: Minimal, focused on explaining non-obvious logic
- **Variable Naming**: Consistent with Unity C# conventions

## Content Organization
- **Modular Structure**: Each feature is self-contained
- **Cross-References**: Liberal use of internal linking
- **Progressive Disclosure**: Basic to advanced concepts
- **Practical Focus**: Real-world game development scenarios
- **Troubleshooting**: Common issues addressed in FAQ sections

## GitBook Specific Elements
- Video embeds for complex procedures
- Hint boxes for important warnings and tips
- Content references for related documentation
- Table of contents integration via SUMMARY.md
